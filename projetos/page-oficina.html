Land Page da Oficina

Situação

A Leilla Noivas, uma loja de vestidos de noiva, enfrentava dificuldades na gestão de estoque devido ao controle manual em papel. Essa prática resultava em imprecisões, lentidão e potenciais perdas, prejudicando a organização e o planejamento da loja.

Tarefa

O objetivo era desenvolver um sistema de estoque digital simples e eficiente para a Leilla Noivas, visando otimizar a gestão das peças de roupa. O sistema deveria permitir o cadastro, consulta, atualização e exclusão de peças, facilitando o controle do estoque.

Ação

-- Um grupo de estudantes do curso de Análise e Desenvolvimento de Sistemas desenvolveu um sistema web utilizando HTML, CSS e PHP.

-- O sistema adotou o modelo CRUD (Criar, Ler, Atualizar e Deletar) para a gestão das peças de roupa.

-- Foi utilizado o banco de dados MySQL para armazenar as informações das peças.

Funcionalidades:

-- Cadastro de peças com informações detalhadas.

-- Consulta de estoque com filtros e busca.

-- Atualização de dados de peças cadastradas.

-- Exclusão de peças do estoque.

Resultado

A implementação do sistema de estoque trouxe diversos benefícios para a Leilla Noivas:

Precisão:

-- Redução de erros e imprecisões no controle de estoque.

Agilidade:

-- Otimização do tempo dos funcionários na consulta e atualização de informações.

Melhoria na Gestão:

-- Fornecimento de informações precisas para a tomada de decisões e planejamento de compras.

Conclusão

O uso de tecnologias web como HTML, CSS e PHP permitiu a criação de um sistema acessível e de fácil utilização. A adoção do modelo CRUD simplificou a gestão das peças de roupa, tornando o sistema intuitivo para os funcionários da loja. A utilização de um banco de dados MySQL garantiu a segurança e a integridade das informações do estoque.
